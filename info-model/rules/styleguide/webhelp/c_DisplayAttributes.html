Display Attributes

Display attributes affect the way in which an element is rendered in your final output.
Name Description Available Values Required
frame Specifies if a border should surround the element and on what sides. top | bottom | topbot | all | sides | none No
scale Specifies a percentage used to resize fonts or images in proportion to their normal size. This attribute is used primarily for print outputs. As a best practice, resize images within the source file rather than relying on this attribute. 50 | 60 | 70 | 80 | 90 | 100 | 110 | 120 | 140 | 160 | 180 | 200 No
outputclass Specifies that a special styling be used during output processing.   No
rev Indicates a specific revision level or tracks a change to a specific change request or problem report. In addition to allowing you to generate reports that show all changes in the entire repository that are related to a specific change request or problem report, this attribute can also be used to flag changes in the final output so that readers can easily scan for changes to the last version; for example, all new and changed content added since the last release might be formatted with change bars.
Define the value of the attribute as the version and release of the product; for example rev="v1.r1". When users view topics that contain flagged content, they have the option to select a specific revision number and see the changes for that revision highlighted within the topic.
Note: Do not use the @rev attribute for minor changes such as editorial issues that are unrelated to a change in the product.
